# Voice Chat Complete Workflow Documentation

## ğŸ¯ System Overview

The voice chat system enables **real-time bidirectional voice communication** between the user and an AI assistant. It uses WebSocket for low-latency audio streaming with Speech-to-Text (STT) and Text-to-Speech (TTS) capabilities.

### Key Components
1. **VoiceChatController** - Main orchestrator
2. **MicStreamer** - Captures and streams user audio
3. **TtsPlayer** - Plays AI voice responses
4. **VoiceActivityDetector (VAD)** - Detects speech vs silence
5. **BargeInDetector** - Detects user interruptions
6. **VoiceWsClient** - WebSocket communication helper

## ğŸ› Common Issues & Solutions

### Issue 1: "No message sent but audio detected"
**Cause:** VAD is suppressing frames during AI playback
**Solution:** This is correct behavior (echo cancellation)

### Issue 2: Duplicate STT messages
**Cause:** Server sends same text multiple times
**Solution:** Implemented deduplication (check _lastFinalText + timestamp)

### Issue 3: Mic permission denied
**Cause:** User denied microphone access
**Solution:** Proper permission handling with clear error messages

### Issue 4: WebSocket disconnects
**Cause:** Network issues or server restart
**Solution:** onResumed() auto-reconnects when screen becomes active

### Issue 5: Audio echo/feedback
**Cause:** Mic picks up AI voice from speaker
**Solution:** VAD suppression during TTS playback + 500ms delay

### Issue 6: Audio crash on Android
**Cause:** FlutterSound recorder not properly closed
**Solution:** 200-300ms delays between stop/start, proper cleanup
